# Slidev 演示系统管理需求文档

## 文档信息

- **版本**: 1.0
- **日期**: 2026-04-16
- **状态**: 草案

## 1. 引言

### 1.1 目的

本文档定义 Slidev 演示系统的管理需求，规范演示内容的创建、组织和维护流程。

### 1.2 范围

本需求适用于：
- slidev/ 目录下的所有演示项目
- Slidev 演示的开发、构建和发布流程

### 1.3 术语定义

- **Slidev**: 基于 Markdown 的演示工具
- **演示主题**: slidev/ 下的单个演示项目
- **Deck**: Slidev 演示文稿

## 2. 总体描述

### 2.1 项目背景

当前项目包含 19+ 个 Slidev 演示主题，涵盖多种技术（Flutter, WebAssembly, WebGL, TailwindCSS 等）。需要统一管理以提高维护效率。

### 2.2 目标用户

- 技术分享者
- 内容创作者
- 学习者

### 2.3 使用场景

- 技术会议演讲
- 团队内部分享
- 个人学习笔记整理

## 3. 功能需求

### 3.1 需求 1: 演示创建标准化

#### 3.1.1 描述

规范化 Slidev 演示的创建流程。

#### 3.1.2 功能需求

- **RF-001**: 提供演示模板（`pnpm create slidev`）
- **RF-002**: 统一目录命名规范
- **RF-003**: 标准化 `package.json` 配置
- **RF-004**: 配置开发服务器端口（避免冲突）

#### 3.1.3 数据需求

- **RD-001**: 目录名使用短横线分隔（kebab-case）
- **RD-002**: 每个演示包含完整的 `frontmatter` 配置

### 3.2 需求 2: 主题和样式管理

#### 3.2.1 描述

统一演示的视觉风格和主题配置。

#### 3.2.2 功能需求

- **RF-005**: 支持自定义主题
- **RF-006**: 统一配色方案
- **RF-007**: 提供通用布局模板

### 3.3 需求 3: 内容管理

#### 3.3.1 描述

规范演示内容的组织和更新。

#### 3.3.2 功能需求

- **RF-008**: 使用 Markdown 编写内容
- **RF-009**: 代码示例使用代码块语法
- **RF-010**: 支持 Mermaid 图表
- **RF-011**: 支持 LaTeX 数学公式
- **RF-012**: 支持图片资源管理

### 3.4 需求 4: 构建和导出

#### 3.4.1 描述

规范演示的构建和发布流程。

#### 3.4.2 功能需求

- **RF-013**: 支持 `pnpm run dev` 开发模式
- **RF-014**: 支持 `pnpm run build` 构建 SPA
- **RF-015**: 支持 `pnpm run export` 导出 PDF
- **RF-016**: 支持 `pnpm run export:png` 导出图片

### 3.5 需求 5: 预览和部署

#### 3.5.1 描述

提供演示预览和在线部署能力。

#### 3.5.2 功能需求

- **RF-017**: 支持预览模式（`/deploy-website` skill）
- **RF-018**: 支持部署到静态托管平台
- **RF-019**: 支持录制演示视频

## 4. 非功能需求

### 4.1 性能需求

- **NFR-001**: 开发服务器启动时间 < 5 秒
- **NFR-002**: 导出 PDF 时间 < 30 秒（50 页内）

### 4.2 可用性需求

- **NFR-003**: 代码示例可交互编辑（Monaco）
- **NFR-004**: 支持演讲者笔记
- **NFR-005**: 支持快捷键导航

### 4.3 可维护性需求

- **NFR-006**: 内容模块化，易于更新
- **NFR-007**: 资源文件统一管理

### 4.4 兼容性需求

- **NFR-008**: 导出 PDF 保持格式一致
- **NFR-009**: 浏览器兼容性（Chrome, Firefox, Safari）
